What are your initial impressions of how `Unit Testing` affects your programming?
It is very helpful for testing small parts of the code without having the program stop executing unless specified. It is also helpful with breaking down which methods or parts of the code work and which don't so you don't have to rewrite entire chunks of code for one specific error. 
What worked, what didn't, what advice would you give someone taking this course in the future?
Make sure to only try to have one assert per test and to breakdown each thing that might go wrong. Such as breaking up the constructor into parts and trying to not tackle everything at once. The code follows a logical order and if you take time to look through that and break it down piece by piece the testing becomes much more comfortable and familiar.
